Put a dollar value on technical debt. Then fix the highest ROI first.

Eunice is a GitLab-first agent workflow that detects technical debt, pulls real repository activity signals (commits, MR review times, issues, pipelines), and produces transparent cost + ROI guidance. It creates actionable issues instead of just saying “this is complex.”

Setup

1. Install

Run the installer in the target GitLab repo. It adds .gitlab/duo configs and eunice.yml.

2. Configure

Add NIA_API_KEY; optionally add Slack tokens. Edit eunice.yml assumptions (rates/thresholds).

3. Trigger

Open an MR. Eunice analyzes, comments with evidence, and creates issues for high ROI debt.

Quick commands
git add .gitlab/ eunice.yml && git commit -m "feat: add eunice" && git push

Methodology (Transparent by Design)

Measured from GitLab
  • Commits touching a file (time-windowed)
  • MR review/merge duration
  • Bug issue time tracking (windowed)
  • Pipeline failures and durations
Configured assumptions
  • Developer hourly rate
  • CI debug hours / compute cost
  • Fix effort estimates
  • Issue creation thresholds
Important note

Eunice combines measured repository signals with user-configured assumptions. It is a prioritization tool, not a forensic accounting system. The value is in transparent decision support and faster refactoring triage.
